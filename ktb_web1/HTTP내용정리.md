| 단어명 | 설명 |
|--------|------|
| FastAPI | 파이썬 기반의 고성능 비동기 웹 프레임워크로, REST API 개발에 적합하다. |
| pyproject.toml | Python 프로젝트의 의존성과 설정을 통합 관리하는 표준 설정 파일. |
| Gunicorn | Python 웹 애플리케이션을 실제 서버 환경에서 실행시키는 WSGI 서버(주로 FastAPI, Django 등과 함께 사용). |
| HTTP Message | 클라이언트와 서버 간 요청(Request)과 응답(Response)을 주고받을 때 사용하는 메시지 구조. |
| HTTP Request Method | 서버에 요청의 **의도**를 알리는 방식 (GET, POST, PUT, DELETE 등). |
| HTTP Status Code | 요청 처리 결과를 숫자로 표현한 코드 (예: 200 OK, 404 Not Found 등). |
| HTTP URL | 인터넷 자원의 위치를 나타내는 주소. |
| Query string | URL 끝에 `?key=value` 형식으로 데이터를 전달하는 문자열. |
| Path Variable | URL 경로 중 일부를 변수처럼 사용하여 데이터 전달하는 방식 (예: `/users/{id}`). |
| JSON | 데이터를 구조적으로 표현하는 경량 포맷 (JavaScript Object Notation). |
| REST API | 자원을 HTTP 방식으로 주고받는 규칙 기반의 API 설계 방식. |
| 디자인패턴 | 반복되는 문제를 해결하기 위한 소프트웨어 설계의 일반화된 해법. |
| Database | 데이터를 체계적으로 저장하고 관리하는 시스템. |
| SQL | 관계형 데이터베이스를 조작하기 위한 표준 언어. |
| 데이터 정의어 (DDL) | 테이블 구조를 생성·변경·삭제하는 명령어 (CREATE, ALTER 등). |
| 데이터 조작어 (DML) | 데이터 삽입·조회·수정·삭제를 수행하는 명령어 (INSERT, SELECT, UPDATE, DELETE 등). |
| 데이터 제어어 (DCL/TCL) | 데이터 접근 권한이나 트랜잭션을 제어하는 명령어 (GRANT, COMMIT 등). |
| JOIN | 여러 테이블을 공통 컬럼으로 연결하여 데이터를 조회하는 연산. |
| 정규화 | 데이터를 규칙 있게 정리하여 중복을 줄이고 일관성을 높이는 과정. |
| 암호화 | 중요한 데이터를 알아볼 수 없도록 숨기는 기술. |
| bcrypt | 비밀번호를 안전하게 암호화하여 저장하기 위한 해시 함수 방식. |
| stateful | 서버가 클라이언트의 상태(누구인지, 어디까지 했는지)를 기억하는 방식. |
| stateless | 서버가 클라이언트의 상태를 기억하지 않고, 각 요청을 독립적으로 처리하는 방식. |
| connectionless | 요청할 때만 잠깐 연결하고, 응답 후 연결을 끊는 통신 방식 (예: HTTP). |
| 인증 · 인가 | **인증**: 누구인지 확인하는 과정. **인가**: 해당 사용자가 그 작업을 할 권한이 있는지 확인하는 과정. |
| 쿠키 | 브라우저가 잠깐 저장하는 작은 데이터 조각(로그인 상태, 최근 본 상품 등). |
| 세션 | 서버 또는 스토리지에 저장되는 사용자 로그인/상태 정보. |
| 프레임워크 | 애플리케이션을 만들기 위한 기본 구조·뼈대를 제공하는 도구. |
| 라이브러리 | 특정 기능을 수행하기 위한 도구 모음. |
| API | 프로그램끼리 기능을 주고받도록 연결해 주는 인터페이스. |
| 페이징 | 데이터가 많을 때 여러 페이지로 나누어 조회하는 기능. |
| HTTPS | 더 안전한 HTTP. 통신 내용을 암호화하여 중간에서 훔쳐보거나 변조하기 어렵게 만든 프로토콜. |
